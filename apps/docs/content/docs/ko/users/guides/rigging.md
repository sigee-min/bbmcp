# 리깅 가이드

리깅 품질은 대부분 계층 설계에서 결정됩니다. 트리가 명확하면 애니메이션 변환이 예측 가능하고, 평면 구조나 임시 부모 연결이 많으면 나중에 움직임이 깨지는 문제가 반복됩니다.

항상 루트 본부터 시작해 몸통, 머리, 팔다리 순으로 하위 구조를 확장하세요. 부모가 없는 본을 대량으로 먼저 만들기보다, 부모-자식 관계를 즉시 확정하면서 쌓는 방식이 안전합니다.

## 운영 원칙

- 모든 비루트 본은 반드시 기존 본을 부모로 가져야 합니다.
- 본 이름은 애니메이션 단계까지 고려해 안정적으로 유지합니다.
- `Parent bone not found`가 나오면 생성 순서를 먼저 점검합니다.
- `invalid_state_revision_mismatch`가 나오면 `get_project_state`로 최신 revision을 읽고 재시도합니다.

리깅을 서둘러 넘어가면 애니메이션 품질이 급격히 떨어집니다. 텍스처보다 먼저 리깅 안정화를 끝내는 것이 전체 비용을 줄이는 지름길입니다.
