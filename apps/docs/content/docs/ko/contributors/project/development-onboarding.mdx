# 개발 온보딩

이 가이드는 일상 개발과 PR 검증을 위한 기여자 진입점입니다.

## 1) 로컬 설정

```bash
git clone https://github.com/sigee-min/ashfox.git
cd ashfox
npm install
```

## 2) 빌드 명령

```bash
npm run build
```

앱별 빌드:

```bash
npm run build:plugin-desktop
npm run build:ashfox
```

스캐폴드 앱 실행:

```bash
npm run dev:gateway
npm run dev:worker
```

## 3) 테스트 및 품질 명령

```bash
npm run test:unit
npm run test:conformance
npm test
```

커버리지 및 품질 게이트:

```bash
npm run test:cov
npm run quality
```

정적 품질 점검:

```bash
npm run quality:check
npm run quality:deadcode
npm run quality:audit
npm run quality:coverage
```

## 4) Docs 빌드

```bash
cd apps/docs
npm ci
npm run build
```

## 5) Web 앱 빌드

```bash
cd apps/web
npm install
npm run build
```

## 6) Docker 스캐폴드

```bash
cd deploy
docker compose up --build
```

## 7) 기여 워크플로우

1. `main`에서 브랜치를 생성합니다.
2. 변경 범위를 분리합니다(runtime, contracts, docs, CI).
3. PR 오픈 전 필수 점검을 실행합니다.
4. PR 설명에 테스트 증거를 포함합니다.
5. 동작 변경 시 스키마와 문서를 함께 갱신합니다.

관련 참고:

- `CONTRIBUTING.md`
- `CODE_OF_CONDUCT.md`
- `SECURITY.md`
- `SUPPORT.md`

## 8) 릴리즈 워크플로우

릴리즈 자동화는 `.github/workflows/release.yml`의 수동 실행으로 동작합니다.

- `publish`: 품질/빌드 검증 후 GitHub Release 발행

워크플로우 실행 전에는 공개 체크리스트를 완료하세요.

- `/ko/docs/contributors/project/release-public-checklist`
