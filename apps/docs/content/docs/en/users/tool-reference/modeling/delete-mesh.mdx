## Overview
- Tool name: `delete_mesh`
- Category: Modeling
- Strict schema: `true`

## Tool Description
Deletes a mesh by id/name (or ids/names for bulk removal).

## Input Schema (JSON)
```json
{
  "type": "object",
  "additionalProperties": false,
  "properties": {
    "id": {
      "type": "string"
    },
    "name": {
      "type": "string"
    },
    "ids": {
      "type": "array",
      "minItems": 1,
      "items": {
        "type": "string"
      }
    },
    "names": {
      "type": "array",
      "minItems": 1,
      "items": {
        "type": "string"
      }
    },
    "ifRevision": {
      "type": "string"
    },
    "includeState": {
      "type": "boolean"
    },
    "includeDiff": {
      "type": "boolean"
    },
    "diffDetail": {
      "type": "string",
      "enum": [
        "summary",
        "full"
      ]
    }
  }
}
```

## Field Reference
| Field | Type | Required | Description | Constraints |

| `id` | `string` | ❌ | - | - |
| `name` | `string` | ❌ | - | - |
| `ids` | `array<string>` | ❌ | - | minItems: 1 |
| `names` | `array<string>` | ❌ | - | minItems: 1 |
| `ifRevision` | `string` | ❌ | - | - |
| `includeState` | `boolean` | ❌ | - | - |
| `includeDiff` | `boolean` | ❌ | - | - |
| `diffDetail` | `enum("summary", "full")` | ❌ | - | enum: "summary", "full" |

## Minimal Request Example
```json
{
  "name": "mesh"
}
```

## Response Example
```json
{
  "structuredContent": {
    "id": "value",
    "name": "value",
    "deleted": [

        "name": "value"
      }
    ]
  }
}
```

## Operational Notes
- Use `ifRevision` for mutation calls when available.
- Do not send fields outside the declared schema.
- Run `validate` before `export` for release safety.
