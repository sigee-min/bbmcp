# 비전 폴백 가이드

기본 경로는 `render_preview`, `read_texture` 응답의 이미지를 클라이언트가 직접 소비하는 방식입니다. 하지만 일부 환경에서는 이미지 블록을 안정적으로 처리하지 못하므로 파일 기반 폴백이 필요합니다.

이 경우 `saveToTmp: true`로 스냅샷을 저장하고, 클라이언트가 지원하는 업로드 경로로 수동 전달합니다. 폴백은 예외 처리 경로로 두고, 가능한 경우 직접 이미지 전달 경로를 우선 유지하는 것이 좋습니다.

스냅샷은 `<project_root>/.ashfox/tmp` 아래에 저장됩니다. 임시 파일은 업로드 후 즉시 정리해 오래된 파일 참조나 저장 공간 누적을 방지하세요.
