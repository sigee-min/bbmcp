## 빠른 요약
최근 실행 흐름의 진단 로그를 내보내 문제 분석에 활용합니다.

## 개요
- 도구 이름: `export_trace_log`
- 카테고리: 프로젝트
- Strict schema: `true`

## 도구 설명
최근 실행 흐름의 진단 로그를 내보내 문제 분석에 활용합니다.

## 입력 스키마 (JSON)
```json
{
  "type": "object",
  "additionalProperties": false,
  "properties": {
    "mode": {
      "type": "string",
      "enum": [
        "auto",
        "writeFile",
        "export"
      ]
    },
    "destPath": {
      "type": "string"
    },
    "fileName": {
      "type": "string"
    }
  }
}
```

## 필드 레퍼런스
| Field | Type | 필수 | 설명 | 제약 |
| --- | --- | --- | --- | --- |
| `mode` | `enum("auto", "writeFile", "export")` | ❌ | 도구 실행 방식을 선택합니다. | enum: "auto", "writeFile", "export" |
| `destPath` | `string` | ❌ | 산출물 저장 경로를 지정합니다. | 선언된 타입 외 추가 제약은 없습니다. |
| `fileName` | `string` | ❌ | 내보내기 파일 이름을 지정합니다. | 선언된 타입 외 추가 제약은 없습니다. |

## 최소 요청 예시
```json
{}
```

## 응답 예시
이 응답은 스키매틱 예시입니다. 실제 값과 선택 필드는 프로젝트 상태 및 런타임 버전에 따라 달라질 수 있습니다.

```json
{
  "structuredContent": {}
}
```

## 운영 노트
- 입력 스키마에 선언된 필드만 전송하세요.
- 응답 예시는 스키매틱이므로 선택 필드는 방어적으로 처리하세요.
- 이 도구는 읽기 전용이므로 일반적으로 `ifRevision`이 필요하지 않습니다.
