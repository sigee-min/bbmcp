# 공개 릴리즈 체크리스트

릴리즈 또는 주요 공개 업데이트 전, 아래 항목을 점검하세요.

## 1. 품질 게이트

- [ ] `npm install`
- [ ] `npm run build`
- [ ] `npm test`
- [ ] `npm run quality`
- [ ] 실패하거나 스킵된 핵심 테스트가 없는지 확인

## 2. 동작 검증

- [ ] 핵심 MCP 플로우를 E2E로 검증:
  - [ ] 프로젝트 라이프사이클 (`ensure_project`, `get_project_state`, `validate`)
  - [ ] 모델링 (`add_bone`, `add_cube`, `update_cube`)
  - [ ] 텍스처링 (`assign_texture`, `paint_faces`, `paint_mesh_face`)
  - [ ] 애니메이션 (`create_animation_clip`, `set_frame_pose`)
  - [ ] 프리뷰/내보내기 (`render_preview`, `export`)
- [ ] 변경 도구의 revision 가드(`ifRevision`) 동작 확인
- [ ] 변경 이후 뷰포트 갱신 동작 확인

## 3. 문서/스키마 정합성

- [ ] `README.md` 및 로컬라이즈드 README가 현재 동작을 반영
- [ ] `apps/docs/content/docs/en/guides/`가 현재 payload/제약을 반영
- [ ] MCP 스키마와 예시가 구현과 일치
- [ ] 브레이킹 체인지 및 마이그레이션 노트가 문서화됨

## 4. 저장소 위생

- [ ] 디버그 로그, 임시 파일, 로컬 전용 아티팩트 제거
- [ ] 생성 파일 최신 상태 확인
- [ ] 라이선스 및 거버넌스 문서 존재 확인:
  - [ ] `LICENSE`
  - [ ] `SECURITY.md`
  - [ ] `CONTRIBUTING.md`
  - [ ] `CODE_OF_CONDUCT.md`
  - [ ] `SUPPORT.md`

## 5. 릴리즈 메타데이터

- [ ] 변경 로그/릴리즈 노트에 다음이 요약됨:
  - [ ] 신규 기능
  - [ ] 수정 사항
  - [ ] 알려진 제한 사항
- [ ] 필요한 앱의 버전만 갱신됨 (`apps/*/package.json`)
- [ ] `.github/workflows/release.yml` 실행 후 자동 생성 릴리즈 노트 확인
- [ ] 클린 환경에서 최종 스모크 테스트 완료
