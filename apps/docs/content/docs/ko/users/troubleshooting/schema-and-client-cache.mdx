## 주요 증상

- 클라이언트 payload가 서버 스키마에서 거부됨
- 업데이트 이후 기존 요청 포맷이 동작하지 않음

## 해결 절차

1. `list_capabilities`로 `toolRegistry.hash`를 확인합니다.
2. hash 변경 시 클라이언트의 도구 스키마 캐시를 갱신합니다.
3. 지원되지 않는 추가 필드를 payload에서 제거합니다.

## 운영 권장사항

- 정적 스키마 하드코딩 대신 `list_capabilities`를 기준으로 동작시킵니다.
