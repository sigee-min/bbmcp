## 빠른 요약
모델/텍스처/UV 규칙을 점검해 릴리즈 전 품질 이슈를 찾습니다.

## 개요
- 도구 이름: `validate`
- 카테고리: 프로젝트
- Strict schema: `true`

## 도구 설명
모델/텍스처/UV 규칙을 점검해 릴리즈 전 품질 이슈를 찾습니다.

## 입력 스키마 (JSON)
```json
{
  "type": "object",
  "additionalProperties": false,
  "properties": {
    "includeState": {
      "type": "boolean"
    },
    "ifRevision": {
      "type": "string"
    }
  }
}
```

## 필드 레퍼런스
| Field | Type | 필수 | 설명 | 제약 |
| --- | --- | --- | --- | --- |
| `includeState` | `boolean` | ❌ | 응답에 최신 프로젝트 스냅샷을 포함합니다. | 선언된 타입 외 추가 제약은 없습니다. |
| `ifRevision` | `string` | ❌ | 낙관적 동시성 제어를 위한 리비전 토큰입니다. | 선언된 타입 외 추가 제약은 없습니다. |

## 최소 요청 예시
```json
{}
```

## 응답 예시
이 응답은 스키매틱 예시입니다. 실제 값과 선택 필드는 프로젝트 상태 및 런타임 버전에 따라 달라질 수 있습니다.

```json
{
  "structuredContent": {
    "findings": [
      {
        "code": "value",
        "message": "value",
        "severity": "error"
      }
    ]
  }
}
```

## 운영 노트
- 입력 스키마에 선언된 필드만 전송하세요.
- 응답 예시는 스키매틱이므로 선택 필드는 방어적으로 처리하세요.
- 이 도구는 읽기 전용이므로 일반적으로 `ifRevision`이 필요하지 않습니다.
