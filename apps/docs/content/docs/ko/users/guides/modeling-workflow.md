# 모델링 워크플로

모델링 단계의 목적은 빠르게 결과를 만드는 것이 아니라, 이후 텍스처와 애니메이션 단계에서 흔들리지 않는 구조를 만드는 것입니다. 한 번에 크게 변경하기보다 작은 단위로 확정하면서 진행해야 revision 충돌과 복구 비용이 줄어듭니다.

권장 순서는 프로젝트 컨텍스트를 먼저 고정한 뒤 본 계층을 만들고, 그 위에 큐브/메시를 점진적으로 올리는 방식입니다. 구조가 안정되기 전에 텍스처 세부 작업으로 들어가면 다시 모델링할 때 재작업이 크게 발생합니다.

## 권장 순서

1. `ensure_project`로 작업 포맷과 프로젝트 컨텍스트를 확정합니다.
2. `add_bone`으로 계층 뼈대를 먼저 잡습니다.
3. `add_cube`를 의미 단위로 나누어 추가합니다.
4. 비율 조정은 `update_bone`, `update_cube`로 반복합니다.
5. 주요 단계마다 `validate`, `render_preview`로 상태를 점검합니다.

모든 변경 호출에는 `ifRevision`을 포함하세요. 이 습관이 멀티 에이전트/재시도 환경에서 가장 큰 안정성을 제공합니다.
