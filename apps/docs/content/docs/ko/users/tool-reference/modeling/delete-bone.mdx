## 개요
- 도구 이름: `delete_bone`
- 카테고리: 모델링
- Strict schema: `true`

## 도구 설명
Deletes a bone by id/name (or ids/names for bulk removal).

## 입력 스키마 (JSON)
```json
{
  "type": "object",
  "additionalProperties": false,
  "properties": {
    "id": {
      "type": "string"
    },
    "name": {
      "type": "string"
    },
    "ids": {
      "type": "array",
      "minItems": 1,
      "items": {
        "type": "string"
      }
    },
    "names": {
      "type": "array",
      "minItems": 1,
      "items": {
        "type": "string"
      }
    },
    "ifRevision": {
      "type": "string"
    },
    "includeState": {
      "type": "boolean"
    },
    "includeDiff": {
      "type": "boolean"
    },
    "diffDetail": {
      "type": "string",
      "enum": [
        "summary",
        "full"
      ]
    }
  }
}
```

## 필드 레퍼런스
| Field | Type | 필수 | Description | Constraints |
| --- | --- | --- | --- | --- |
| `id` | `string` | ❌ | - | - |
| `name` | `string` | ❌ | - | - |
| `ids` | `array<string>` | ❌ | - | minItems: 1 |
| `names` | `array<string>` | ❌ | - | minItems: 1 |
| `ifRevision` | `string` | ❌ | - | - |
| `includeState` | `boolean` | ❌ | - | - |
| `includeDiff` | `boolean` | ❌ | - | - |
| `diffDetail` | `enum("summary", "full")` | ❌ | - | enum: "summary", "full" |

## 최소 요청 예시
```json
{
  "name": "bone"
}
```

## 응답 예시
```json
{
  "structuredContent": {
    "id": "value",
    "name": "value",
    "removedBones": 0,
    "removedCubes": 0,
    "deleted": [
      {
        "name": "value"
      }
    ]
  }
}
```

## 운영 노트
- 변경 도구는 `ifRevision` 사용을 권장합니다.
- 스키마에 없는 필드는 보내지 마세요 (strict schema).
- `validate`로 최종 상태를 확인한 후 `export`를 수행하세요.
