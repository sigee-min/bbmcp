# 모델링/텍스처 반복 작업

## 목적
모델링과 텍스처 수정 사이클을 빠르게 반복하면서 결과를 확인합니다.

## 사전 준비
- Blockbench가 설치되어 있고 로컬 플러그인 로드가 가능해야 합니다.
- 저장소 루트에서 플러그인 번들을 빌드해야 합니다 (`npm run build:plugin-desktop`).
- 플러그인 설정에서 MCP 엔드포인트 경로(`/mcp`)를 확인해야 합니다.

## 빠른 시작
1. `project.ensure_project`로 프로젝트 컨텍스트를 준비합니다.
2. 모델링 계열 도구(`add_*`, `update_*`, `delete_*`)를 실행합니다.
3. 텍스처 읽기/할당(`read_texture`, `assign_texture`)을 수행합니다.

## 상세 단계
1. `project.ensure_project`로 프로젝트 컨텍스트를 준비합니다.
2. 모델링 계열 도구(`add_*`, `update_*`, `delete_*`)를 실행합니다.
3. 텍스처 읽기/할당(`read_texture`, `assign_texture`)을 수행합니다.
4. `project.render_preview`로 중간 결과를 점검합니다.

## 실패와 복구
- 연결 실패: 엔드포인트 경로, 서비스 실행 상태, 방화벽/권한 설정을 재확인합니다.
- 스키마 오류: 최소 요청으로 재시도하고 클라이언트 스키마 캐시를 갱신합니다.
- 검증 실패: 리포트된 항목을 수정한 뒤 Quick Start부터 다시 실행합니다.
