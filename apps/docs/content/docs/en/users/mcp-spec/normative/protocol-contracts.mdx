# Protocol Contracts

## MUST

- Keep MCP request/response structure schema-compatible across patch releases.
- Preserve deterministic request IDs per call path.
- Keep capability discovery available for client bootstrap.

## SHOULD

- Use explicit error codes for retryable vs non-retryable failures.
- Keep compatibility notes for schema changes in release notes.
