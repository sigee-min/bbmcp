## 목표

서비스 준비 상태, 엔드포인트 연결성, 기본 클라이언트 상호운용성을 확인합니다.

## 첫 운영 점검 흐름

1. gateway와 worker 서비스를 실행합니다.
2. 엔드포인트 경로와 네트워크 접근성을 확인합니다.
3. 외부 클라이언트에서 `system.list_capabilities`를 호출합니다.
4. 읽기 전용 프로젝트 호출을 한 번 수행합니다.
5. revision-safe 변경 호출을 한 번 실행하고 응답 처리를 확인합니다.

기준 점검이 끝나면 Ashfox 워크플로우 가이드와 작업 플레이북으로 진행하세요.
