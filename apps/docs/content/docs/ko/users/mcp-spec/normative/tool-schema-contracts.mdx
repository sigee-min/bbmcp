# Tool Schema Contracts

## MUST

- 기존 도구의 필수 필드명과 의미를 유지합니다.
- 잘못된 페이로드는 검증 결과와 함께 명확히 거부합니다.
- 공개된 enum/discriminator 값은 안정적으로 유지합니다.

## SHOULD

- 필수 필드 변경 대신 선택 필드 확장을 우선합니다.
- 엄격한 제약 추가 전 호환성 영향도를 문서화합니다.
