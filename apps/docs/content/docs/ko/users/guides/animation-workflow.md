# 애니메이션 워크플로

Ashfox 애니메이션은 고수준 자동화보다 명시적 제어를 우선합니다. 클립 정의, 포즈 타임라인, 트리거 타임라인을 분리해 관리하면 문제 발생 시 원인 분리가 쉬워지고 재현성이 높아집니다.

애니메이션 단계는 모델 구조와 텍스처 의도가 충분히 안정된 뒤에 들어가는 것이 좋습니다. 구조가 흔들리는 상태에서 키프레임을 먼저 많이 쌓으면 후반 수정 비용이 커집니다.

## 권장 순서

1. 프로젝트 상태와 revision을 먼저 확인합니다.
2. `create_animation_clip`으로 클립 길이, loop, fps를 명확히 정합니다.
3. `set_frame_pose`를 시간 순서대로 누적합니다.
4. 포즈 타이밍이 안정된 뒤 `set_trigger_keyframes`를 추가합니다.
5. `render_preview`로 시각 검증 후 내보내기로 넘어갑니다.

프레임은 fps 기준 시간으로 환산되므로, 클립 생성 단계의 fps 선택이 이후 전체 결과를 좌우합니다.
