# 텍스처/UV 동작 원칙

이 문서는 스키마 나열보다 실제 운영 기준을 설명하는 데 초점을 둡니다. 목적은 팀 구성원이 동일한 기준으로 페인팅 요청을 만들고, 오류 상황을 같은 방식으로 해석하도록 맞추는 것입니다.

Ashfox는 per-face UV 전제를 사용하며 UV 도구는 내부에서 처리됩니다. 사용자는 페인트 의도와 요청 구조를 안정적으로 유지하는 데 집중해야 합니다.

## 운영 시 고정해야 할 기준

1. UV overlap은 오류로 간주합니다.
2. UV scale mismatch도 오류로 간주합니다.
3. 기본 밀도는 `uvPixelsPerBlock`에서 시작합니다.
4. 페인팅 요청은 단일 대상 + 단일 연산 구조를 유지합니다.

다중 대상/다중 연산을 한 번에 넣는 방식은 디버깅 비용이 높고 실패 원인을 흐립니다. 작은 단위로 요청을 나누는 방식이 품질과 재현성에 유리합니다.
