## Overview
- Tool name: `export`
- Category: Project
- Strict schema: `true`

## Tool Description
Exports the current project to the requested output format (including native codec targets like glTF/OBJ/FBX when available).

## Input Schema (JSON)
```json
{
  "type": "object",
  "required": [
    "format",
    "destPath"
  ],
  "additionalProperties": false,
  "properties": {
    "format": {
      "enum": [
        "java_block_item_json",
        "gecko_geo_anim",
        "animated_java",
        "generic_model_json",
        "gltf",
        "native_codec",
        "auto"
      ]
    },
    "codecId": {
      "type": "string"
    },
    "destPath": {
      "type": "string"
    },
    "includeState": {
      "type": "boolean"
    },
    "ifRevision": {
      "type": "string"
    }
  }
}
```

## Field Reference
| Field | Type | Required | Description | Constraints |

## Minimal Request Example
```json
{
  "format": "java_block_item_json",
  "destPath": "./out/model.json"
}
```

## Response Example

```json
  }
}
```

## Operational Notes
- Use `ifRevision` for mutation calls when available.
- Do not send fields outside the declared schema.
- Run `validate` before `export` for release safety.
