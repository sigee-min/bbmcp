# LLM 텍스처 전략

LLM 텍스처링은 한 번에 많이 칠하는 방식보다, 의도를 명확히 나눈 짧은 반복 루프에서 품질이 높게 나옵니다. 핵심은 요청 수를 줄이는 것이 아니라, 한 요청이 하나의 시각적 결정을 담도록 만드는 것입니다.

## 권장 루프

1. `assign_texture`로 텍스처 컨텍스트를 고정합니다.
2. `paint_faces` 또는 `paint_mesh_face`로 단일 의도를 반영합니다.
3. `render_preview`로 결과를 확인하고 다음 연산을 결정합니다.

프롬프트에는 "이번 호출에서 만들고 싶은 시각 효과"를 먼저 명시하고, 베이스 색상, 경계, 하이라이트를 단계적으로 분리하세요. 이 방식이 수정과 롤백이 가장 쉽습니다.

검증 단계에서 UV 관련 오류가 발생하면 페인트를 계속 누적하지 말고 상태를 먼저 안정화한 뒤 다시 칠해야 불필요한 재작업을 줄일 수 있습니다.
