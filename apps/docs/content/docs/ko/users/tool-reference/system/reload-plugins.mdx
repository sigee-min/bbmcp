## 빠른 요약
플러그인 런타임을 재로드해 변경된 확장 로직을 즉시 반영합니다.

## 개요
- 도구 이름: `reload_plugins`
- 카테고리: 시스템
- Strict schema: `true`

## 도구 설명
플러그인 런타임을 재로드해 변경된 확장 로직을 즉시 반영합니다.

## 입력 스키마 (JSON)
```json
{
  "type": "object",
  "required": [
    "confirm"
  ],
  "additionalProperties": false,
  "properties": {
    "confirm": {
      "type": "boolean"
    },
    "delayMs": {
      "type": "number"
    }
  }
}
```

## 필드 레퍼런스
| Field | Type | 필수 | 설명 | 제약 |
| --- | --- | --- | --- | --- |
| `confirm` | `boolean` | ✅ | 파괴적 동작에 대한 명시적 확인 플래그입니다. | 선언된 타입 외 추가 제약은 없습니다. |
| `delayMs` | `number` | ❌ | `delayMs` 요청 파라미터입니다. | 선언된 타입 외 추가 제약은 없습니다. |

## 최소 요청 예시
```json
{
  "confirm": true
}
```

## 응답 예시
이 응답은 스키매틱 예시입니다. 실제 값과 선택 필드는 프로젝트 상태 및 런타임 버전에 따라 달라질 수 있습니다.

```json
{
  "structuredContent": {
    "scheduled": true,
    "delayMs": 0,
    "method": "devReload"
  }
}
```

## 운영 노트
- 입력 스키마에 선언된 필드만 전송하세요.
- 응답 예시는 스키매틱이므로 선택 필드는 방어적으로 처리하세요.
- 변경 도구에서 지원되는 경우 `ifRevision`을 사용해 충돌을 줄이세요.
