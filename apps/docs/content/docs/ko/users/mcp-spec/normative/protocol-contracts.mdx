# Protocol Contracts

## MUST

- 패치 릴리즈 범위에서 MCP 요청/응답 구조의 스키마 호환성을 유지합니다.
- 호출 경로별 요청 ID를 결정적으로 유지합니다.
- 클라이언트 부트스트랩을 위한 capability 조회를 항상 제공합니다.

## SHOULD

- 재시도 가능/불가능 오류를 명확한 코드로 구분합니다.
- 스키마 변경 시 호환성 메모를 릴리즈 노트에 남깁니다.
